RAG · 校园知识库

浙科大校园智能问答助手 v3（campus_assistant_v3）

基于通义千问（DashScope）与Chroma向量库的校园专属助手：支持多格式文档入库、对话问答，并在回答中隐式依赖检索结果、面向师生口吻自然输出。

  • 产品形态：Gradio 搭建聊天主界面，独立 Tab 支持 PDF / TXT / MD 批量上传并刷新向量库，可一键清空会话上下文。
  • 检索链路：查询理解模块生成检索计划；可选查询改写后做双路检索，再按来源与内容去重合并，控制上下文长度。
  • 生成策略：结合问题类型（事实类 / 流程类 / 规章类等）切换提示词与回答风格，限制胡编、要求「资料不足则明确说明」。
  • 工程结构：服务层 CampusAssistantService 串联初始化、问答、上传；子模块涵盖 retrieval、answering、query_understanding、会话存储与上传管理，便于单测与迭代。

技术栈

Python Gradio LangChain 生态 / Chroma 通义千问 API 向量嵌入
Bot · 企业 IM

钉钉 Stream 本地指令机器人（dingtalk_agent）

基于 钉钉 Stream 长连接的群聊 / 单聊机器人：在收到指定指令时，以白名单 + 超时 + 输出截断的方式执行本机命令，适合个人自动化与运维辅助实验。

  • 消息协议：注册 Chatbot 与 Delegate 两类 topic，兼容不同投递路径；解析文本后识别 ping 与 run: 前缀指令。
  • 安全边界：仅允许配置中的命令前缀（如 python、pytest、dir、echo 等），subprocess 带超时与退出码回传，回复长度上限可配。
  • 运行时：启动前校验 AppKey / AppSecret；Windows 下切换 Selector 事件循环策略缓解 Proactor 与 WebSocket 不稳定；外层循环断线自动重连。
  • 配置：轻量 .env 加载，日志级别与允许前缀均可通过环境变量调整。
